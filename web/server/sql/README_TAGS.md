# Добавление тегов к репозиториям

## 1. Добавить колонку tags в БД

Выполните SQL скрипт:
```bash
mysql -u your_user -p your_database < server/sql/add_tags_column.sql
```

Или вручную:
```sql
ALTER TABLE repos ADD COLUMN tags TEXT NULL COMMENT 'Comma-separated tags for search';
ALTER TABLE repos ADD FULLTEXT INDEX idx_tags (tags);
```

## 2. Формат хранения тегов

Теги хранятся как строка с разделителями-запятыми:
```
библиотека,http,rest,api
```

## 3. Добавление тегов к репозиториям

### Вручную через SQL:
```sql
UPDATE repos SET tags = 'библиотека,http,rest,api' WHERE id = 1;
UPDATE repos SET tags = 'тестирование,bdd,cucumber' WHERE id = 2;
UPDATE repos SET tags = 'парсинг,xml,json' WHERE id = 3;
```

### Через скрипт импорта:
При импорте данных добавьте поле `tags` в JSON:
```json
{
  "id": 1,
  "name": "oscript-http",
  "tags": "библиотека,http,rest,api"
}
```

## 4. Как это работает

- **Поиск**: Теги автоматически включены в поиск (name, description, author, tags)
- **Отображение**: Теги показываются только в модальном окне репозитория
- **Таблица**: Теги НЕ отображаются в основной таблице

## 5. Рекомендации по тегам

- Используйте строчные буквы
- Разделяйте запятыми без пробелов (или с пробелами после запятой)
- Избегайте дублирования информации из других полей
- Примеры хороших тегов:
  - Технологии: `rest`, `soap`, `graphql`, `websocket`
  - Назначение: `тестирование`, `отчеты`, `интеграция`, `парсинг`
  - Фреймворки: `bdd`, `cucumber`, `junit`
  - Форматы: `xml`, `json`, `yaml`, `csv`

## 6. Примеры использования

### Поиск по тегам:
- Пользователь вводит "rest" → находит все репозитории с тегом "rest"
- Пользователь вводит "тестирование" → находит все тестовые фреймворки

### Просмотр тегов:
- Клик на репозиторий → открывается модальное окно
- В модальном окне отображаются все теги репозитория
